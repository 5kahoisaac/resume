  repo's immutability principle, but inputs are always freshly-parsed objects,
  so no live bug. Not worth a plan.
- **Constant session token with no per-session identity / revocation**
  (`auth.ts:17`): by design for a single-user tool. Noted as a deferred
  follow-up inside plan 003, not a standalone finding.
- **Empty `catch {}` guards in `pdf.ts` / `RichTextEditor.tsx`**: deliberate,
  comment-documented best-effort error handling — not silent-failure bugs.
- **Duplicated `escapeHtml` (linkify.ts + storage.ts) and palette-resolution

  Deliberately not planned to keep the batch focused; fold into a future cleanup
  if desired.
